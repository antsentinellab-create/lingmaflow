## Context

LingmaFlow v0.2.1 已經完整實現了執行引擎和 CLI 工具，但 README.md 仍然是簡短版本。這導致：
- 新用戶無法快速上手完整功能
- 缺少 OpenSpec 整合的實戰範例
- 已知問題沒有文檔化，重複踩坑

## Goals / Non-Goals

**Goals:**
- 提供完整的雙語（中英）README 文檔
- 包含所有 v0.2.1 功能的詳細說明
- 新增 OpenSpec 整合使用範例
- 收錄實戰中的已知問題與解決方案
- 保持專業且易讀的風格

**Non-Goals:**
- 不改變現有功能或 API
- 不創建新的教程或示例代碼
- 不修改其他文檔文件（AGENTS.md, TASK_STATE.md 等）

## Decisions

### 文檔結構決策
- **採用模塊化章節**: 將 README 分為清晰的章節（安裝、快速開始、核心概念、CLI 命令、OpenSpec 整合、已知問題等）
- **雙語並行**: 每個章節同時提供中文和英文版本，方便國際用戶
- **實戰導向**: 強調實際使用場景和故障排除，而非純理論說明

### 內容組織決策
- **已知問題獨立成章**: 將實戰經驗整理為「問題 - 症狀 - 原因 - 解決」格式
- **程式碼區塊標註語言**: 所有代碼示例明確標註 bash/python 等語言
- **版本標註清晰**: 在相關功能旁標註引入版本（v0.2.1）

## Risks / Trade-offs

**[Risk]** README 過於冗長可能嚇跑快速瀏覽的用戶  
→ **Mitigation**: 在開頭提供精簡的「快速開始」章節，詳細內容放在後面章節

**[Risk]** 雙語內容導致文件過長  
→ **Mitigation**: 使用清晰的標題層級，方便用戶跳轉到需要的語言部分

**[Risk]** 已知問題可能隨版本過時  
→ **Mitigation**: 在每個問題後標註適用版本，並定期更新文檔
